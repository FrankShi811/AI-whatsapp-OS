# AI 模型与 WhatsApp 商机分析对齐（1.11.0）

## 当前已确定的产品契约

1. AI Provider 采用 DeepSeek 或 OpenAI Chat Completions 兼容 HTTPS API。
2. 设置页调用 `GET <Base URL>/models`，使用 Windows 凭据中的 API Key 拉取模型目录；模型目录、拉取时间和用户选择保存在本机，API Key 不写入 SQLite。
3. 新客户与从未成功完成 AI 分析的客户初始为 `D / 0`。Excel 导入、重复覆盖和人工编辑不会自行计算 A/B/C 等级。
4. WhatsApp Inbox 收到已关联客户的新回复后：
   - 先保存原始消息；
   - 捕捉报价、数量、交期、样品、明确需求、积极、延后和异议等检索信号；
   - 将客户置为“等待 AI”；
   - 使用当前选中模型串行分析 CRM 字段和最近 WhatsApp 上下文；
   - 只有通过结构校验的结果才回写评分、等级、阶段、画像、证据、下一步动作和风险。
5. 关键词不直接加分，也不直接改变等级；模型必须回看原文语境。Provider 失败时保留客户和消息，标记为可重试。
6. AI 分析结果只影响商机智能、Dashboard 等级分布以及依赖这些权威 CRM 字段的筛选/模板，不修改 WhatsApp 原始消息。

## 暂时保留、等待后续模板替换的部分

- 当前继续使用八因素输出契约及 100 分边界：市场价值 15、公司规模 10、产品匹配 20、采购能力 15、回复积极度 15、活跃时间 10、明确需求 10、注册或咨询 5。
- 等级边界暂为 A≥80、B=60–79、C=40–59、D<40。
- 收到用户提供的新评分标准和商机智能模板后，只替换评分契约、提示词和展示字段，不改动模型发现、WhatsApp 证据链、排队与失败恢复机制。
