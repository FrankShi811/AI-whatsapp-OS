# WAFlow 个人账号 WhatsApp 自动化执行计划

## 风险声明

该连接器基于非官方 WhatsApp Web 多设备协议，不属于 Meta/WhatsApp Business Platform。账号可能被要求重新验证、限制或登出，协议更新也可能导致连接器失效。WAFlow 不提供规避风控、设备指纹伪装、代理轮换或绕过账号限制的能力。

Baileys 官方仓库也明确要求自行承担风险，并反对垃圾消息和批量自动消息。WAFlow 因此只对已导入、号码有效、未退订且被用户明确纳入 Campaign 的个人联系人发送，并强制保留人工审批、间隔、每日上限、暂停和审计。

## 运行形态

- `WAFlow-Windows-x64.exe`：WPF 原生 UI、客户与 Campaign 数据。
- `WAFlow.WhatsApp.Bridge`：本地 Baileys 桥，负责 QR、多设备会话和消息事件。
- 两者使用标准输入输出 JSON-RPC，不开放 HTTP 端口。桥接程序以独立 Node SEA Windows EXE 形式嵌入主程序，运行时按版本释放到本地应用目录。
- 会话文件保存在当前 Windows 用户的 `%LOCALAPPDATA%\WAFlow\whatsapp-sessions`，内容使用 AES-256-GCM 加密；每个账号的随机密钥独立保存在 Windows 凭据管理器。

8. T8.7：真实二维码事件验收、EXE 发布和最终原生界面验收。真实账号扫码与测试联系人发送留给用户在自有账号上完成，交付测试不触碰真实消息。

## 实施状态（v1.2.0）

- T8.0–T8.6 已完成。
- T8.7 已完成打包桥接 EXE 的真实二维码事件、加密状态写入、主 EXE 发布、Inbox/Campaign 原生界面和正常退出验证。
- 交付测试未扫描真实个人账号，也未发送真实 WhatsApp 消息；首次账号连接与首条许可联系人发送需要用户自行验收。

## 发送保护

- 默认只允许个人会话，不自动向群组、状态或频道发送。
- Campaign 创建后必须人工审批才能排期。
- 号码无效、已退订、重复号码或消息为空时禁止入队。
- 固定间隔用于业务排期，不实现用于躲避平台检测的行为随机化。
- 收到登出、连接限制或连续失败时立即暂停全部任务。
- 每个收件人持久化幂等键，重启后不重复发送。
