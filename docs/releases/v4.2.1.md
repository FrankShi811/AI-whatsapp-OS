# AI Sales OS v4.2.1

## WhatsApp 真实发送确认修复

- 消息发送必须取得 WhatsApp 服务端消息 ID，并校验实际目标号码；无法确认目标时保留“待确认”或显示失败，不再创建本地伪成功记录。
- 送达和已读回执只接受属于实际收件人的 `userJid`，忽略本机设备、自身账号及号码不匹配回执，防止错误绿勾和送达时间。
- 手工会话、AI 回复、后台同步、Customer Success Agent 与自动化群发统一采用同一发送确认契约。
- 未确认消息不会推进 CRM 阶段、写入有效学习样本或计入自动化任务成功数。
- 新增消息 ID 缺失、目标验证失败、回执号码错配与缺失状态回归测试。

## 数据与发布边界

- 不删除、不覆盖客户、消息历史、WhatsApp 账号、邮件、自动化任务、AI 分析或配置。
- 本版本仅通过 GitHub Release 发布 Windows 中文安装包与 Velopack 自动更新资产。
- 不覆盖或重启当前电脑上的正式程序；macOS 构建继续暂停。
