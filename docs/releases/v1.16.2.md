# AI Sales OS 1.16.2

## WhatsApp 群发真实性修复

- 群发任务不再把“Bridge 已受理请求”直接统计为成功，只有收到 WhatsApp 服务器发送确认后才计入成功。
- WhatsApp 异步返回发送错误时，任务收件人立即改为失败，历史表的成功数、失败数、完成进度和成功率同步更新。
- 发送前校验号码是否已注册 WhatsApp；未注册号码不会尝试触达，并显示明确失败原因。
- 90 秒内没有收到服务器确认的消息不计为成功，且为避免重复触达不会自动重发。

## 历史数据修复

- 启动时自动比对群发收件人与已保存的 WhatsApp 消息回执。
- 旧版本中误记为成功、但实际消息状态为失败的记录会自动更正；客户、会话和任务原始数据不会删除。

## 交付与数据安全

- Windows 与 macOS 更新继续通过 GitHub Release 分发。
- 更新不会覆盖 `%LOCALAPPDATA%\WAFlow\waflow.db`、WhatsApp 会话、API 配置或客户报告。
