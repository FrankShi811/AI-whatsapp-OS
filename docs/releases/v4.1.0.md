# AI Sales OS v4.1.0

发布日期：2026-07-24

## DHgate Customer Success Conversation Agent

本版本把 WhatsApp 单会话 AI 助手升级为以全局客户为中心的 DHgate Customer Success Conversation Agent。Agent 的固定身份是 DHgate 客户成功经理：理解与澄清采购需求、补齐采购要素、协调人工跟进并维护客户关系；它不会冒充商户、工厂或供应商，也不会对库存、最终价格、生产、物流、海关、退款、赔偿或平台政策作未经验证的承诺。

## 跨 WhatsApp 账号客户连续性

- 同一客户在多个 WhatsApp 账号下使用一个全局 `customer_id`、Customer Brain、Customer 360、时间线、采购需求和待办。
- 每个账号保留独立 Persona 与关系记忆，回复只使用当前发送账号的身份。
- 身份解析按人工绑定、精确 JID、已确认 E.164、已确认别名、国家辅助唯一匹配和唯一完整数字体推断的顺序执行。
- 歧义、无匹配和冲突不会自动创建、合并或回复客户；姓名、公司和邮箱只作为人工候选。
- 跨账号人工确认关系、主账号和身份审计在应用重启后继续保留。

## Agent 状态、安全与人工交接

- 新增 `AUTO_OFF`、`SUGGEST_ONLY`、`COPILOT_ACTIVE`、`AUTO_ACTIVE`、`IDENTITY_RESOLUTION_REQUIRED`、`HUMAN_REQUIRED`、`HUMAN_ACTIVE`、`RESUME_REVIEW` 八种状态。
- 同一全局客户只能有一个自动回复账号；其他账号会被全局锁阻断，防止重复或冲突回复。
- 高风险请求、退款/补偿承诺、提示注入和秘密索取会立即保存客户原话、发送一次同语言占位回复、进入全局人工交接并冻结全部关联账号。
- 人工接管、解决、恢复审核和自动账号切换均写入不可丢失的审计记录；受限状态不会因重启、同步或新消息自动恢复。

## 采购需求五要素

采购需求固定收集：

1. 产品；
2. 规格；
3. 数量；
4. 目标价格与币种；
5. 目的地。

每项占完整度 20%，且必须绑定真实客户消息证据。跨账号出现冲突时保留新旧值和来源，不覆盖既有事实；只有人工确认后才会成为当前采购需求。

## 产品联动

- WhatsApp Inbox 客户抽屉显示身份解析、全局客户、关联账号、Agent 模式、采购五要素、冲突和人工交接。
- Dashboard Today Brief 汇总身份待确认、人工交接、采购完成及跨账号跟进。
- Customer Brain、CRM、Lead Intelligence、任务和客户事件读取同一全局客户事实；AI 提议保持建议性质，不覆盖人工锁定字段。
- AI Provider 失败时保存可重试状态并保留最近一次有效判断。

## 数据升级与回滚

- 迁移只新增表、索引和兼容列，不删除或覆盖原客户、消息、账号、Brain、Lead Intelligence、任务、邮件、Campaign 或报告。
- 首次启动以幂等方式回填现有明确 CRM/WhatsApp 关系；重复启动不会复制数据。
- 旧版本会忽略新增旁路表，因此代码可直接回滚到上一 GitHub Release。
- 用户数据继续保存在 `%LOCALAPPDATA%\WAFlow\waflow.db`，Windows 安装和自动更新不会覆盖该文件。

## 发布边界

本版本仅发布 Windows 中文安装包和 Velopack 自动更新资产，不覆盖开发电脑根目录的正式 `AI Sales OS.exe`。macOS 构建继续暂停，等待 Developer ID、签名与公证条件恢复。
